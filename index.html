Advanced concepts — extra credit
Not bolted on — each concept traces directly back to Assignment 1 decisions.
SAFe architectural runway
The modular add-on architecture in Project 1 was designed to avoid blocking Project 2. The recommendation engine microservice is decoupled from the ML scoring engine — Project 2 plugs in without touching Project 1 code. The feedback loop service begins collecting outcome data in Project 1 so that by the time Project 2 starts, there is enough training signal. This is what SAFe calls "runway" — the architecture is already prepared for the next horizon before you get there.
Hybrid delivery model
Not all modules are delivered the same way. Bank API integration and the compliance layer are planned predictively — requirements are clear, low risk. The recommendation engine and notification UX are delivered adaptively — early user feedback shapes which trigger conditions and copy perform best. We document this split so the team knows which sprints are flex and which are fixed.
DevOps integration planning
Three practices built in: (1) Feature flags — recommendations roll out to 5% of users first, metrics measured before full release; (2) CI/CD pipeline — new model versions tested against a holdout set before deployment, zero-downtime releases; (3) Automated compliance testing — recommendation copy is linted against CFPB language rules on every commit. Connects back to Assignment 1 success criteria — we need to release fast and measure against churn and conversion targets.
All three concepts trace back to Assignment 1 — the two-project program structure, the adaptive/hybrid lifecycle, and the MOV metrics. This assignment extends the same thinking technically.
